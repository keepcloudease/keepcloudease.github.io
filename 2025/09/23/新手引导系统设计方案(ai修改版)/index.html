新手引导系统设计方案

恒逸2025-09-232026-01-18

1. 设计概述

本系统旨在通过配置驱动的方式，引导玩家快速熟悉游戏核心功能。系统采用模块化设计，通过监听游戏全局事件和玩家行为，动态触发引导逻辑，并确保客户端与服务器的数据强一致性。

核心原则

  • 配置驱动：逻辑与代码分离，策划通过 GuideConfig 定义触发条件与表现。
  • 断点续传：基于 guideStep 状态管理，支持掉线重连后从当前步骤继续。
  • 模块化：对话、点击、地图操作等逻辑解耦，易于横向扩展。
  • 强同步：关键步骤完成即时同步服务器，防止外挂或数据丢失。

3. 核心模块详解

3.1 核心控制器 (GuideView)

  • 初始化 (Initialization)：注册 GuideViewCheckExcute 等全局事件。
  • 分发器 (excute)：解析配置表中的 action 字段，通过 switch-case 或反射工厂模式分发给具体的子视图（如 GuideTalkView）。
  • 异常处理 (OnGuideFailed)：处理如“目标UI未打开”、“场景加载未完成”等异常情况，触发 faildJumpId 进行回退或跳过。

3.2 表现层

  • 遮罩层 (Mask)：
    • 实现原理：使用 RaycastTarget 阻挡点击，在目标按钮位置“挖空”或生成高亮区域。
    • 坐标转换：需处理 WorldToScreenPoint 确保遮罩准确覆盖目标 UI。
  • 特效层 (Effect)：
    • 加载手指点击动画、光圈特效等。
  • 文本提示 (Tips)：
    • 动态显示策划配置的引导文案。

3.3 数据与网络

  • 本地状态：PlayerModel.Inst.guideStep 作为客户端真理来源。
  • 网络同步：
    • 步骤更新：GameNetBase.getNet<GuideNet>().SetStep()。
    • 断线重连：登录时服务器下发最新的 guideStep，客户端根据该 ID 自动恢复引导。

4. 扩展性建议

  1. 强弱引导区分：在 GuideConfig 中增加 isForce 字段。强引导通过遮罩屏蔽操作，弱引导仅显示提示但不锁定屏幕。
  2. 条件检查器：增加 Condition 字段（如：等级>10级），在 OnCheckExcute 中预先判断，避免不满足条件时强行触发引导。
  3. 多分支支持：虽然当前是线性 nextId，但可扩展支持 nextIds 数组，根据玩家选择跳转不同分支。
