1. 新手引导系统的设计原理

新手引导系统的核心目标是通过配置驱动的方式，帮助玩家快速熟悉游戏的核心功能和操作流程。它通过监听玩家的行为事件，结合引导配置文件，动态触发引导逻辑，确保玩家能够逐步掌握游戏内容。

系统的主要设计原则包括：

  • 配置驱动：通过配置文件（GuideConfig）定义引导的触发条件、执行逻辑和表现形式。
  • 模块化设计：将不同类型的引导（如对话引导、地图引导、奖励引导等）拆分为独立模块，便于扩展和维护。
  • 状态管理：通过玩家的引导进度（guideStep）记录当前引导状态，支持断线重连和引导恢复。
  • 与后台交互：每完成一个引导步骤都会向服务器同步进度，确保数据一致性。

2. 执行方案

核心类与模块

  • GuideView：引导系统的主控类，负责引导的初始化、流程控制和状态管理。
  • GuideTalkView：处理对话类引导。
  • GuideMapView：处理地图操作类引导。
  • GuideGetRoleView：处理奖励展示类引导。

主要功能点

  1. 引导初始化

    • 在游戏启动时，通过 GuideView.Initialization() 方法初始化引导系统。
    • 注册全局事件监听（如 eNameUI.GuideViewCheckExcute），确保引导系统能够实时响应玩家行为。

  2. 引导触发

    • 通过 OnCheckExcute() 方法检查当前引导步骤（PlayerModel.Inst.guideStep）。
    • 根据步骤 ID 从配置文件中加载对应的引导配置（GuideConfig）。
    • 根据配置的 action 字段，调用不同的引导模块（如 ShowTalkView()、ShowClickUIButton() 等）。

  3. 引导表现

    • 遮罩层：通过 mask 高亮目标区域，引导玩家注意力。
    • 特效层：通过 effect 显示动态特效，增强引导效果。
    • 提示文本：通过 tipsText 显示引导说明。

  4. 引导完成

    • 每完成一个引导步骤，调用 OnGuideComplete() 方法：
      • 更新玩家的引导进度（guideStep）。
      • 向服务器同步进度（GameNetBase.getNet<GuideNet>().SetStep()）。
      • 如果有奖励，调用 ShowGetAward() 请求服务器发放奖励。

  5. 引导失败

    • 如果引导失败，调用 OnGuideFailed() 方法：
      • 跳转到补救引导步骤（faildJumpId）。
      • 同步失败状态到服务器。

3. 运行逻辑

引导流程

  1. 触发条件检测

    • 通过全局事件或玩家行为触发 OnCheckExcute()。
    • 检查当前引导步骤是否满足触发条件。

  2. 加载引导配置

    • 从配置文件中加载当前步骤的引导逻辑（如目标对象、引导类型、表现形式等）。

  3. 执行引导逻辑

    • 根据配置的 action 字段，调用对应的引导模块。
    • 显示遮罩、特效、提示文本等引导内容。

  4. 完成或失败处理

    • 引导完成后，更新进度并同步到服务器。
    • 引导失败时，跳转到补救步骤并同步状态。

5. 总结

这个新手引导系统的特点是：

  1. 配置驱动：通过配置文件灵活定义引导逻辑，便于策划调整。
  2. 模块化设计：不同类型的引导逻辑分离，易于扩展和维护。
  3. 实时交互：通过全局事件监听和服务器同步，确保引导流程的实时性和数据一致性。
  4. 用户友好：通过高亮、特效、提示文本等方式，直观地引导玩家操作。
